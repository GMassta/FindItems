# FindItems
### Прототип игры с поиском предметов

-Из предложенных материалов необходимо собрать работающий уровень/прототип игры в стиле “Найди предмет”<br/>
-Игровой уровень состоит из двух основных частей. Игровая область с поисковыми предметами  и UI интерфейс со списком, что необходимо найти игроку (панель для интерфейса - составляем из стандартных элементов Unity UI, шрифт также берем дефолтный)<br/>
-Для поиска одновременно доступно только 3 предмета.<br/>
-При клике на верный предмет

а) Предмет плавно исчезает на игровой области<br/>
б) Исчезает наименование предмета на UI<br/>
в) Если в игровой области еще остаются предметы для поиска, то на месте предыдущего наименования на UI - появляется название следующего предмета для поиска и он становится доступным для поднятия на игровой области

-При нахождении всех предметов появляется сообщение о прохождении игры (составляется из стандартных элементов Unity UI)

### Дополнительные детали:

-Поместить на сцену таймер на 2 минуты. По истечении времени должно выскакивать окно о проигрыше<br/>
-Добавить возможность выводить список предметов не текстом, а изображениями (в исходниках psb имеется отдельная папка для UI)<br/>
-На кнопку Esc должен производиться выход из игры<br/>
-Написать ScriptableObject с возможностью редактирования уровня

а) Отключать и включать конкретные предметы<br/>
б) Менять режим отображения списка в UI (текст или изображения)<br/>
в) Менять значение таймера или отключать его<br/>
г) Менять порядок предметов

### Обязательные условия при выполнении:

-Версия редактора 6000.2.10f1<br/>
-Наличие DI Container (Zenject  или VContainer на выбор)

### Дополнительные библиотеки

Addressables - гибкое управление ресурсами<br/>
UniTask - асинхронные операции<br/>
DOTween - позволяет быстро создавать анимации<br/>
VContainer - Di контейнер
